# CFUserNotificationCreate(_:_:_:_:_:)

**Framework**: Core Foundation  
**Kind**: func

Creates a CFUserNotification object and displays its notification dialog on screen.

**Availability**:
- macOS 10.0+

## Declaration

```swift
func CFUserNotificationCreate(_ allocator: CFAllocator!, _ timeout: CFTimeInterval, _ flags: CFOptionFlags, _ error: UnsafeMutablePointer<Int32>!, _ dictionary: CFDictionary!) -> CFUserNotification!
```

#### Return Value

The new CFUserNotification object. Ownership follows the [`The Create Rule`](https://developer.apple.comhttps://developer.apple.com/library/archive/documentation/CoreFoundation/Conceptual/CFMemoryMgmt/Concepts/Ownership.html#//apple_ref/doc/uid/20001148-103029).

## Parameters

- `allocator`: The allocator to use to allocate memory for the new object. Pass   or kCFAllocatorDefault to use the current default allocator.
- `timeout`: The time to wait before the notification dialog dismisses itself if the user does not respond. If  , the notification never times out.
- `flags`: A set of flags describing the type of notification to display. These flags specify an alert level for the notification (see  ), determine whether radio buttons or checkboxes are to be used (see  ), specify which, if any, of these buttons are checked by default (see  ), specify whether any of the text fields are to be secure text fields (see  ), and determine which element of a pop-up menu, if present, should be selected by default (see  ). Combine these flags together by performing a bitwise-OR operation with all the individual flags.
- `error`: On return contains an integer error code. If  , the user notification was successfully created and displayed.
- `dictionary`: A description of the elements to display in the notification dialog. The possible keys are listed in  . The dictionary must contain a value for the key  , but the other keys are optional.
