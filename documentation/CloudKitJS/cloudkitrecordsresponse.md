# CloudKit.RecordsResponse

**Framework**: CloudKit JS  
**Kind**: class

A [`CloudKit.RecordsResponse`](cloudkit.recordsresponse.md) object encapsulates the results of fetching records.

**Availability**:
- CloudKit JS 1.0+

## Declaration

```swift
interface CloudKit.RecordsResponse
```

#### Overview

The methods in the [`CloudKit.Database`](cloudkit.database.md) class that operate on records and the [`commit`](cloudkit.recordsbatchbuilder/commit.md) method in the [`CloudKit.RecordsBatchBuilder`](cloudkit.recordsbatchbuilder.md) class return a `Promise` object that resolves to a [`CloudKit.RecordsResponse`](cloudkit.recordsresponse.md) object if the operation is successful. You should not create instances of this class directly.

Errors can occur for individual operations although the overall method may succeed. If an individual operation is successful, a corresponding [`CloudKit.Record`](cloudkit.record.md) dictionary appears in the [`records`](cloudkit.recordsresponse/records.md) array. If an individual operation is unsuccessful, a corresponding [`CKError`](cloudkit/ckerror.md) appears in the errors array.

For examples using the [`CloudKit.Database`](cloudkit.database.md) methods that resolve to a [`CloudKit.RecordsResponse`](cloudkit.recordsresponse.md) object, see the `saveRecord` and `fetchRecord` methods in [`CloudKit.Database`](cloudkit.database.md).
